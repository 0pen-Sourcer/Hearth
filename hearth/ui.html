Auto-boot last model on launch

            
When you start Hearth, automatically load the last local GGUF you used into the built-in server. Saves the 15-30s "wait for me to click Use this" beat. Does NOT affect cloud brains (Grok / Gemini / OpenAI) — those just need the API key.

          

          

        


        

          

            
Default context window · built-in server only

            
Only applies when Hearth runs its own llama.cpp server and loads a GGUF that has no saved config yet. LM Studio / Ollama / cloud brains set their own context — this is ignored for them. Higher = more headroom but more KV-cache VRAM.

          

          
        


        

          

            
Idle auto-unload

            
How long the server can sit idle before unloading the model to free VRAM. 0 = never unload. Same as HEARTH_KEEP_ALIVE_MIN.

          

          
        


        

          

            
Server port

            
Default 1234 to match LM Studio. Change only if you have a conflict.

          

          
        


        

          Default context / port / idle changes apply on the next server start. Hit Restart above to act on the new value now.
        


        


        

        
MCP servers

        

          Hearth speaks the Model Context Protocol — the open standard for letting one
          AI client talk to many tool servers. Hearth itself is an MCP server (point any
          MCP-compatible client at hearth/mcp_server.py to use Hearth's tools elsewhere).
          And below, you can wire other MCP servers into Hearth so their tools
          show up alongside the built-in ones.
        


        

          

            
Outbound — Hearth as a CLIENT

            

              Paste a standard mcp.json config. Each server runs as a subprocess; its tools join the
              Hearth tool list and are usable by every chat brain (local / cloud). Restart Hearth
              after editing — bridge spin-up happens at launch.
            

          

          
        


        

          

            
Active MCP bridges

            

              Live list of MCP servers Hearth has connected to. Each row shows the server name,
              transport (stdio / sse), and how many tools it contributed. Errors surface here so a
              broken config is visible instead of silently dropped.
