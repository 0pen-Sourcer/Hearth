# Make a PPTX

You have the outline + need to produce a `.pptx` file. Use the bundled
`scripts/build_pptx.py` helper — wraps `python-pptx` (already a Hearth
dep, used for read_file on `.pptx`) so the boilerplate stays here.

## Slide outline format

Pass a simple text file where each slide is separated by `---` and the
first line of each slide is the title:

```
Cover slide title
- subtitle line 1
- subtitle line 2

---

Section heading
- Bullet one
- Bullet two
- Bullet three

---

Conclusion
- Key takeaway
```

## Steps

1. **Write the outline.** `write_file` to
   `<workspace>/generated/<slug>.outline.txt` in the format above.
2. **Run the builder.** `run_command`:
   ```
   python <hearth>/skills/make-pptx/scripts/build_pptx.py \
     --outline <workspace>/generated/<slug>.outline.txt \
     --out <workspace>/generated/<slug>.pptx \
     --title "<deck title>"
   ```
   (`<hearth>` = the folder field load_skill returned for this skill.)
3. **Confirm.** Script prints the final path on stdout.
4. **Open it for them.** `run_command('powershell -Command "Invoke-Item ''<full path>''"')` so PowerPoint pops the deck open.

## PICK A STYLE per deck — don't reuse one theme for everything

Use `--style-file <path.json>` (NOT inline `--style`) because PowerShell
ruins the JSON quoting. Workflow: `write_file` a small .json with the
style dict, then pass its path to `--style-file`.

Style keys:
- `accent` / `title_color` — title text color (any hex)
- `body_color` — bullet text color
- `font` — Calibri (default), Arial, Cambria, Georgia, Tahoma
- `title_size` (default 40), `body_size` (default 18)

Topic-appropriate riffs (write each to its own .style.json, vary them):
- **Tech / product pitch**: `{"accent":"#6b4ec7","body_color":"#1a1a24","font":"Calibri"}`
- **Corporate / finance**: `{"accent":"#1f3a5f","body_color":"#222","font":"Cambria","title_size":36}`
- **Bold / startup energy**: `{"accent":"#c4451c","body_color":"#1a1a24","title_size":48}`
- **Academic / sober**: `{"accent":"#222","body_color":"#222","font":"Georgia","title_size":34}`
- **Minimal / monochrome**: `{"accent":"#000","body_color":"#333","font":"Arial"}`

## NO AI-tells in slides
Don't add slides like "Generated by Hearth" or "Compiled <date>". The
deck is the user's deliverable, not a Hearth demo. Reader should never
guess it was AI-built.

## Mechanics
- 16:9 widescreen
- Title slide first, content slides after
- One slide per `---` block

## Don't
- Don't try to embed live data / charts. If the user wants a chart,
  generate a PNG separately (matplotlib + write_file) and call this
  skill with the image referenced in a slide as `image: <path>`.
- Don't author 30-slide decks unprompted — ask the user how many.
